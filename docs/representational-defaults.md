# Representational defaults

Internal contributor guidance for example content, placeholders, and demos. Subtle but intentional — without this, defaults revert culturally.

Related: [design-philosophy-checklist.md](./design-philosophy-checklist.md), `inclusiveTable` in campaign defaults.

## Defaults

| Area | Bias | Avoid |
|------|------|-------|
| Seed/example characters | Feminine and enby names **≥50%**; mixed casts | All-male parties, grimdark dude-party |
| Placeholder copy | Mixed/feminine-leaning (`Mira`, `Sable`, `Juniper`) | `Sir Aldric`, `Lord Darkwood`, generic Hero |
| Onboarding / empty states | Relational narratives, discovery | Combat optimization, war-room framing |
| Screenshots & marketing | Diverse casts; collaborative scenes | All-male grids, tactical-only shots |
| Campaign seeder / templates | Varied leadership gender in orgs/families | Patriarchal feudalism as only template |
| LFG / recruitment examples | Lead with `inclusiveTable` and inclusive GM tags | Lone-wolf combat as hero example |
| Plugin slot demos | Same mix as core examples | Copying grimdark defaults |

## Cursor rule

New example content, placeholders, and demo data should **default feminine/enby-forward unless context-specific**.
