# Environment Variables

Canonical reference for every supported Esiana environment variable.

**Docker self-hosting:** copy [`.env.example`](../../.env.example) to `.env` and set `POSTGRES_PASSWORD` and `JWT_SECRET`. Everything else has defaults in [`docker-compose.yml`](../../docker-compose.yml) or is optional (see below).

**Local Node development:** use [`backend/.env.example`](../../backend/.env.example) and [`frontend/.env.example`](../../frontend/.env.example).

---

## Docker Compose — required in `.env`

| Name | Default | Required | Description | Example |
|------|---------|----------|-------------|---------|
| `POSTGRES_PASSWORD` | — | **Yes** | PostgreSQL password. Used by the `postgres` service and embedded in the app `DATABASE_URL` compose generates. | `openssl rand -hex 32` |
| `JWT_SECRET` | — | **Yes** | Secret for signing session tokens. Use a long random value in production. | `openssl rand -hex 32` |

---

## Docker Compose — optional in `.env`

| Name | Default | Required | Description | Example |
|------|---------|----------|-------------|---------|
| `PUBLIC_ORIGIN` | `http://localhost:8080` (matches `COMPOSE_HTTP_PORT`) | Optional | Public browser URL (no trailing slash). Sets `FRONTEND_ORIGIN`, `CORS_ORIGIN`, and `BACKEND_PUBLIC_ORIGIN`. Required when using a reverse proxy — see [Reverse Proxies.md](Reverse%20Proxies.md). | `https://esiana.example.com` |
| `ESIANA_VERSION` | `latest` | Optional | GHCR image tag. Pin for pull-based upgrades. | `v1.0.1` |
| `AUTH_SECRETS_KEY` | (empty) | Optional | 32-byte base64 AES key for encrypting OIDC client secrets in Admin. Required when using Identity Providers in production. | `openssl rand -base64 32` |
| `OPENAPI_DOCS_ENABLED` | `true` | Optional | `true` exposes `/api/docs` in production; `false` hides Swagger on public hosts. | `false` |
| `COMPOSE_HTTP_PORT` | `8080` | Optional | Host port mapped to the esiana container. | `8080` |
| `TRUST_PROXY` | `false` | Optional | `true` when behind a reverse proxy that sets `X-Forwarded-*` headers. | `true` |
| `COOKIE_SECURE` | `false` | Optional | `true` for HTTPS-only session cookies. Set when serving only over TLS. | `true` |
| `ENABLE_INTERNAL_NGINX` | `true` | Optional | `true` runs internal nginx (web delivery layer: SPA + API proxy on container port 80). `false` is API-only on port 3001 — SPA not served. | `false` |

## Docker Compose — set in compose (not `.env`)

| Name | Default | Description |
|------|---------|-------------|
| `POSTGRES_USER` | `esiana` | PostgreSQL role. Fixed in compose; do not duplicate in `.env`. |
| `POSTGRES_DB` | `esiana` | PostgreSQL database name. Fixed in compose. |
| `DATABASE_URL` | `postgresql://esiana:${POSTGRES_PASSWORD}@postgres:5432/esiana` | Generated by compose from fixed user/db and `POSTGRES_PASSWORD`. Do not set in `.env` unless overriding for advanced setups. |
| `NODE_ENV` | `production` | Runtime mode for the container. |
| `UPLOADS_DIR` | `/data/uploads` | Upload storage inside the container (backed by the `uploads` volume). |
| `PLUGINS_DIR` | `/app/plugins` | Plugin packages inside the container (backed by the `plugins` volume). |
| `ESIANA_PRODUCT_VERSION` | (from image build) | Read-only product version metadata (matches root `package.json` at image build). Observability only — app reads version from `/app/package.json`. |
| `ENABLE_INTERNAL_NGINX` | `true` | `true` runs nginx web delivery layer on port 80; `false` is API-only. |
| `TRUST_PROXY` | `false` | `true` when behind a reverse proxy — see [Reverse Proxies.md](Reverse%20Proxies.md). |

## Backend — core runtime

Defined in [`backend/src/config/env.ts`](../../backend/src/config/env.ts).

| Name | Default | Required | Description | Example |
|------|---------|----------|-------------|---------|
| `NODE_ENV` | `development` | Optional | `production` in Docker; `development` enables verbose Prisma logging. | `production` |
| `PORT` | `3001` | Optional | HTTP listen port (internal; nginx proxies in Docker). | `3001` |
| `DATABASE_URL` | `postgresql://esiana:esiana@localhost:5432/esiana` | Optional (dev) | Prisma connection string. Set by compose in Docker. | `postgresql://esiana:secret@localhost:5432/esiana` |
| `DATABASE_PROVIDER` | `postgresql` | Optional | Read into app config; must match `provider` in `schema.prisma`. | `postgresql` |
| `JWT_SECRET` | `dev-secret-change-me` | **Yes (prod)** | Session token signing secret. | long random string |
| `JWT_EXPIRES_IN` | `7d` | Optional | Session token lifetime. | `7d` |
| `COOKIE_NAME` | `esiana_token` | Optional | Session cookie name. | `esiana_token` |
| `COOKIE_SECURE` | `false` | Optional | `true` for HTTPS-only cookies. Set `true` behind TLS in production. | `true` |
| `COOKIE_SAME_SITE` | `lax` | Optional | `strict`, `lax`, or `none`. | `lax` |
| `CORS_ORIGIN` | `http://localhost:5173` | Optional | Allowed frontend origin for API CORS. Set by compose in Docker. | `https://esiana.example.com` |
| `TRUST_PROXY` | `false` | Optional | `true` when behind a reverse proxy. | `true` |
| `UPLOADS_DIR` | `../uploads` | Optional | Local filesystem storage root (relative to `backend/`). | `/data/uploads` |
| `PLUGINS_DIR` | `../plugins` | Optional | Runtime plugin package directory. | `/app/plugins` |
| `STORAGE_PROVIDER` | `filesystem` | Optional | Active storage driver ID. | `filesystem` |
| `STORAGE_REDIRECT_THRESHOLD_BYTES` | `5242880` (5 MB) | Optional | Prefer redirect delivery above this size. | `5242880` |
| `OPENAPI_DOCS_ENABLED` | (unset) | Optional | `true` / `false`; when unset, `/api/docs` enabled in non-production only. | `true` |

Product version (admin UI, update checks, plugin `engines.esiana-core` gate) comes from root `package.json` `version` at build/runtime — not from an environment variable.
