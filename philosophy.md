# Product Philosophy

## Core Identity

Esiana is not a VTT.
Esiana is campaign narrative infrastructure.

The focus is:
- continuity
- historical state
- collaborative worldbuilding
- knowledge revelation
- long-term campaign memory
- interconnected lore systems

The project intentionally avoids:
- combat automation
- tactical simulation
- initiative systems
- line-of-sight systems
- rules adjudication engines

## Design Priorities

1. Narrative continuity over combat tooling
2. Knowledge organization over rules execution
3. Temporal world state over static wiki pages
4. Player discovery over omniscient data exposure
5. Extensible systems over hardcoded assumptions

# Anti-Goals

Esiana is not intended to become:

- a tactical combat simulator
- a Foundry replacement
- a rules automation engine
- a virtual tabletop
- a character optimization platform
- a live-service MMO campaign host

Feature proposals should be evaluated against these anti-goals.
